EDUNEXUS PROJECT DOCUMENTATION
=============================

1. PROJECT OVERVIEW
-------------------
EduNexus is a full-stack education platform with role-based access for students and teachers.
Core capabilities include:
- Authentication and role-based protected routes
- Quiz creation (manual + AI)
- Quiz attempt and result evaluation
- Teacher mark review workflow
- Leaderboard and gamified eco points
- Study material publishing and requests
- Group collaboration and AI notes
- News feed, daily facts, and daily games

2. TECH STACK
-------------
Backend
- Node.js (ES modules)
- Express
- MongoDB + Mongoose
- JWT authentication
- OpenAI SDK (AI quiz/material/fact generation)
- Axios (news API fetch)
- Multer (file uploads)
- node-cron (periodic news refresh)

Frontend
- React 19 + Vite
- React Router
- Tailwind + custom CSS design system

5. RUNNING THE PROJECT
----------------------
Backend
1. cd backend
2. npm install
3. cp .env.example .env
4. fill real credentials
5. npm run dev

Frontend

3. cp .env.example .env

6. AUTHENTICATION AND SESSION MODEL
-----------------------------------
- Auth endpoints:
  - POST /api/auth/register
  - POST /api/auth/login
  - GET  /api/auth/profile
- JWT token and user are stored on frontend in localStorage via session service:
  - key: edunexus_token
  - key: edunexus_user
- ProtectedRoute enforces:
  - user must be logged in
  - role must match route role requirement

10. CORE DATA FLOWS
-------------------
A) Login flow
- user submits credentials
- frontend POST /api/auth/login
- token + user stored in session service
- route redirects to role dashboard

B) Student quiz flow
- student opens dashboard -> GET /api/quiz
- opens attempt page -> GET /api/quiz/:quizId
- submits answers -> POST /api/results/submit
- redirected to result detail -> GET /api/results/detail/:resultId

C) Teacher review flow
- teacher opens quiz results -> GET /api/results/:quizId
- opens single submission detail
- edits marks -> PUT /api/results/review
- refreshed detail fetched again

D) Materials flow
- teacher uploads -> POST /api/materials
- student/teacher browse -> GET /api/materials
- student requests AI material -> POST /api/materials/request

E) Groups flow
- teacher creates group and adds member
- teacher uploads group material
- student/teacher request AI notes for topic

11. SECURITY NOTES
------------------
- Never commit production API keys or connection strings.
- Rotate keys immediately if leaked.
- Keep backend/.env out of version control.
- Validate and sanitize all user inputs server-side.
- Add rate limiting and request validation for production.
